## 提示词

```
我 fork 了 DoTheBetter/aria2_build，现在需要基于它做以下定制修改。请帮我实现：

## 目标

在保留原项目所有能力的基础上，增加用户可配置的构建参数，让 workflow_dispatch 支持自定义输入。

## 具体修改

### 1. 修改 workflow 文件，添加 workflow_dispatch 输入参数

在原有的 `on` 触发器中，添加 `workflow_dispatch.inputs`，支持以下参数：

```yaml
on:
  workflow_dispatch:
    inputs:
      aria2_ref:
        description: "Aria2 源码版本（tag 如 1.37.0，或 branch 如 master）"
        required: false
        default: "master"
      target:
        description: "目标平台（留空则构建全部）"
        required: false
        type: choice
        options:
          - all
          - x86_64-unknown-linux-musl
          - aarch64-unknown-linux-musl
          - x86_64-w64-mingw32
          - arm-unknown-linux-musleabi
      enable_zlib_ng:
        description: "使用 zlib-ng 替代 zlib（true/false）"
        required: false
        default: "true"
      enable_libressl:
        description: "使用 LibreSSL 替代 OpenSSL（true/false）"
        required: false
        default: "false"
  schedule:
    - cron: "0 0 * * 6"
  release:
    types:
      - released
```

### 2. 根据 target 参数控制 matrix 构建

当用户选择具体平台时，只构建该平台；选择 all 时构建全部：

```yaml
strategy:
  matrix:
    cross_host: ${{ fromJSON(
      github.event.inputs.target == 'all' || github.event.inputs.target == '' 
      && '["arm-unknown-linux-musleabi","aarch64-unknown-linux-musl","x86_64-unknown-linux-musl","x86_64-w64-mingw32"]'
      || format('["{0}"]', github.event.inputs.target)
    ) }}
```

或者更简单的方式：用 `if` 条件判断是否跳过某个 job。

### 3. 将输入参数传递给 build.sh

```yaml
- name: compile
  env:
    CROSS_HOST: "${{ matrix.cross_host }}"
    USE_LIBRESSL: "${{ github.event.inputs.enable_libressl || '0' }}"
    USE_ZLIB_NG: "${{ github.event.inputs.enable_zlib_ng || '1' }}"
    ARIA2_VER: "${{ github.event.inputs.aria2_ref || '' }}"
  run: |
    chmod +x "${GITHUB_WORKSPACE}/build.sh"
    "${GITHUB_WORKSPACE}/build.sh"
```

注意：`ARIA2_VER` 需要在 build.sh 中处理，如果非空则下载指定版本的 release tarball，否则用 master。

### 4. 在 build.sh 中支持 ARIA2_VER 环境变量

检查 build.sh 中 `build_aria2()` 函数，确保当 `ARIA2_VER` 非空时：
- 下载 `https://github.com/aria2/aria2/releases/download/release-${ARIA2_VER}/aria2-${ARIA2_VER}.tar.gz`
- 而不是下载 master tarball

### 5. 添加 git 身份配置（补丁应用前）

在应用补丁的步骤前，添加：

```yaml
- name: Configure git identity
  run: |
    git config --global user.email "build@aria2-build"
    git config --global user.name "Aria2 Build"
```

### 6. 保持原有功能不变

- 保留 schedule（每周自动构建）
- 保留 release（自动发布）
- 保留 cache（下载缓存）
- 保留 matrix（多平台并行）
- 保留 build_info 生成
- 保留所有 patch 文件

## 注意事项

- 不要删除 build.sh，它是最核心的构建逻辑
- 不要修改 patch 文件
- workflow_dispatch 和 schedule/release 可以共存
- `target` 默认值为空或 all，保证原有定时构建不受影响
```
