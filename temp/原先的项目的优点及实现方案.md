## 项目定位

本项目是 DoTheBetter/aria2_build 的精简定制版，核心优势：**用户可自定义构建参数**。

## 优点清单

### 1. 用户可配置的构建参数（workflow_dispatch inputs）

DoTheBetter 版本固定构建所有平台，无法自定义。本项目通过 `workflow_dispatch` 输入参数，用户可以：

- 选择目标平台（linux-x86_64 / linux-aarch64 / windows-x86_64）
- 指定 aria2 源码版本（branch / tag / commit）
- 控制是否静态编译
- 附加自定义 configure 参数

### 2. 双模式构建（lite + full）

| 模式 | 文件 | 定位 |
|------|------|------|
| lite | `lite.yml` | 轻量快速，用 Ubuntu 系统包 |
| full | `full.yml` | 完整功能，所有依赖从源码编译 |

用户按需选择，不需要等全平台构建完成。

### 3. 定时自动构建

`lite.yml` 每周一 UTC 03:00 自动拉取最新 master 源码构建，保持产物始终最新。

## 关键实现方案

### 一、workflow_dispatch 输入参数定义

```yaml
# lite.yml / full.yml
on:
  workflow_dispatch:
    inputs:
      aria2_ref:
        description: "Aria2 源码分支 / Tag / Commit（留空则使用 master）"
        required: false
        default: "master"
      target:
        description: "目标平台"
        required: false
        default: "linux-x86_64"
        type: choice
        options:
          - linux-x86_64
          - linux-aarch64
          - windows-x86_64
      enable_static:
        description: "是否静态编译（true/false）"
        required: false
        default: "true"
      extra_configure_flags:
        description: "附加 configure 参数（可留空）"
        required: false
        default: ""
  schedule:
    - cron: "0 3 * * 1"  # 每周一 UTC 03:00
```

### 二、参数传递到构建步骤

```yaml
env:
  ARIA2_REF: ${{ github.event.inputs.aria2_ref || 'master' }}
  TARGET: ${{ github.event.inputs.target || 'linux-x86_64' }}
  ENABLE_STATIC: ${{ github.event.inputs.enable_static || 'true' }}
  EXTRA_CONFIGURE_FLAGS: ${{ github.event.inputs.extra_configure_flags || '' }}
```

### 三、lite 模式 — 用系统包快速编译

```yaml
# 安装系统依赖
- name: Install build dependencies
  run: |
    sudo apt-get update
    sudo apt-get install -y \
      build-essential autoconf automake autopoint libtool pkg-config \
      libcppunit-dev libssl-dev zlib1g-dev libc-ares-dev \
      libsqlite3-dev libssh2-1-dev libgmp-dev libgcrypt20-dev \
      gettext git ca-certificates curl

# 克隆源码
- name: Clone latest Aria2 source
  run: |
    git clone --depth 1 --branch "$ARIA2_REF" \
      https://github.com/aria2/aria2.git aria2-src
    echo "ARIA2_COMMIT=$(git -C aria2-src rev-parse --short HEAD)" >> "$GITHUB_ENV"

# 应用补丁（需要先配置 git 身份）
- name: Apply patches
  working-directory: aria2-src
  run: |
    git config --global user.email "build@aria2-build"
    git config --global user.name "Aria2 Build"
    PATCH_DIR="${GITHUB_WORKSPACE}/.github/patches"
    for patch in ${PATCH_DIR}/*.patch; do
      git am --3way "$patch" || git am --3way --reject "$patch" || true
    done

# 根据 target 动态配置 configure 参数
- name: Configure
  working-directory: aria2-src
  run: |
    CONFIGURE_FLAGS="--without-gnutls --with-openssl \
      --without-libxml2 --with-libexpat=no --with-libxml2=no \
      --enable-libaria2"
    case "$TARGET" in
      linux-x86_64)
        CONFIGURE_FLAGS="$CONFIGURE_FLAGS --with-libcares --with-sqlite3 --with-libssh2 --with-libgmp --with-libgcrypt"
        ;;
      linux-aarch64)
        CONFIGURE_FLAGS="$CONFIGURE_FLAGS --with-libcares --with-sqlite3 --with-libssh2 --with-libgmp --with-libgcrypt --host=aarch64-linux-gnu"
        ;;
      windows-x86_64)
        CONFIGURE_FLAGS="$CONFIGURE_FLAGS \
          --without-libcares --without-libssh2 --without-libgmp \
          --without-libgcrypt --without-sqlite3 --without-libz \
          --host=x86_64-w64-mingw32"
        ;;
    esac
    if [ "$ENABLE_STATIC" = "true" ]; then
      CONFIGURE_FLAGS="$CONFIGURE_FLAGS LDFLAGS=-static"
    fi
    ./configure $CONFIGURE_FLAGS $EXTRA_CONFIGURE_FLAGS
```

### 四、full 模式 — 从源码编译所有依赖

```yaml
# 在 Docker 容器中交叉编译
- name: Build Linux (musl)
  container:
    image: abcfy2/musl-cross-toolchain:latest
  env:
    CROSS_HOST: x86_64-unknown-linux-musl
  run: |
    PREFIX="${GITHUB_WORKSPACE}/prefix"
    mkdir -p "$PREFIX"

    # 1. 构建 zlib-ng
    cmake -B build -DCMAKE_SYSTEM_NAME=Linux \
      -DCMAKE_C_COMPILER="${CROSS_HOST}-gcc" \
      -DBUILD_SHARED_LIBS=OFF -DZLIB_COMPAT=ON \
      -DCMAKE_INSTALL_PREFIX="$PREFIX"
    cmake --build build -j$(nproc) && cmake --install build

    # 2. 构建 OpenSSL
    ./Configure linux-x86_64 --prefix="$PREFIX" --static
    make -j$(nproc) CROSS_COMPILE="${CROSS_HOST}-" && make install_sw

    # 3. 构建 aria2
    export LDFLAGS="-L${PREFIX}/lib64 -L${PREFIX}/lib -s -static --static"
    export PKG_CONFIG_PATH="${PREFIX}/lib64/pkgconfig:${PREFIX}/lib/pkgconfig"
    autoreconf -i
    ./configure --host="$CROSS_HOST" --prefix="$PREFIX" \
      --enable-static --disable-shared --without-libxml2 ARIA2_STATIC=yes
    make -j$(nproc)
```

### 五、补丁应用注意事项

```yaml
# 必须先设置 git 身份，否则 git am 会报 "Committer identity unknown"
git config --global user.email "build@aria2-build"
git config --global user.name "Aria2 Build"

# working-directory 和 run 脚本中的 cd 不要重复设置
# 错误示例：
- name: Apply patches
  working-directory: aria2-src  # 已设置工作目录
  run: |
    cd aria2-src                # ❌ 会导致双重嵌套
    git am ...

# 正确示例：
- name: Apply patches
  working-directory: aria2-src  # 已设置工作目录
  run: |
    git am ...                  # ✅ 直接执行
```
